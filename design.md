# AI热点监控与智能防伪平台——系统设计方案 (System Design Document)

## 一、 项目概述

在AI技术日新月异、信息呈爆炸式增长的背景下，海量的行业动态中夹杂着大量的炒作（Hype）、标题党甚至虚假合成内容。本项目旨在构建一个高并发、低延迟的 **AI热点监控与智能防伪平台**。

平台通过自动化爬虫集群，并发抓取海内外多源异构数据（Twitter/X、GitHub、Bilibili等），引入搜索引擎（Google、Bing）实施交叉验证，并依托 **OpenRouter** 对接 **DeepSeek v4 pro / DeepSeek-R1 系列** 模型进行深度推理判别，最终通过极具科技感的响应式 Web 页面，为用户呈现高确定性的行业热点与全天候的定制监控服务。

---

## 二、 核心功能设计

### 1. 每日 10:00 公共热点看板 (Public Hotspot Board)
* **功能定位**：面向全网公开的公共信息看板，每日准时发布最具含金量的 AI 进展。
* **业务流程**：
    1.  **凌晨至早晨（自动化抓取）**：Go 后端定时触发多源爬虫，聚合过去 24 小时全网涉及 AI 的原始数据。
    2.  **09:30（AI 聚类与清洗）**：调用 DeepSeek 提取核心事件，去重、聚类并按热度与真实性评分，筛选出 Top 5 核心热点。
    3.  **10:00（准时上线）**：生成结构化摘要及演进脉络，写入 Redis 缓存。前端主页在 10:00 准时刷新，承载高并发访问。

### 2. 关键词实时监控与 AI 防伪裁判 (Real-time Monitor & Anti-Fake Alert)
* **功能定位**：用户（前期为个人）自定义高敏感度关键词，系统全天候毫秒级监听，并利用 AI 排除伪造/虚假内容。
* **AI 防伪裁判机制**：
    * **一阶段（命中捕获）**：网络爬虫命中用户设置的关键词（如“GPT-5”、“开源SOTA”）后，立即拉取上下文。
    * **二阶段（搜索引擎交叉验证）**：后端并发调用 Google/Bing Search API 搜索该事件，采集主流技术媒体、官方域名或权威代码库的收录状态。
    * **三阶段（DeepSeek 深度判定）**：将原始消息与搜索引擎交叉数据输入 DeepSeek 推理模型，执行逻辑推演，拦截缺乏信源证实、含煽动炒作词汇（如“炸裂”、“超越一切”）的虚假消息。
* **即时通知**：通过判定为“真”的内容，立即通过 WebSocket 或 SSE（Server-Sent Events）推送到 Web 前端，或异步触发 Telegram/邮件通知。

### 3. 领域周期性深挖简报 (Domain Periodic Scanning)
* **功能定位**：针对特定广义领域（如“AI编程”、“多模态生成”）进行长周期的数据积淀与深度总结。
* **灵活配置架构**：
    * 系统默认扫描周期为 **24小时**，但在产品设计上开放配置接口（如可选 1h、6h、12h、24h、48h）。
    * 到达指定周期后，系统将该时间段内收集的所有行业碎片、技术讨论、代码提交记录合并，由 DeepSeek 提炼出一份极具洞察力的“领域结构化深度简报”。

## 三、 系统技术栈架构

为了满足高并发抓取、动态任务流转、高端视觉展示以及低成本高推理性能的要求，技术栈选型如下：

| 分层 | 技术选型 | 核心作用与优势 |
| :--- | :--- | :--- |
| **前端服务** | Next.js (React) | 支持 SSR（服务端渲染），对公共看板的 SEO 友好；拥有极佳的页面流转与首屏加载性能。 |
| **前端样式/动效** | Tailwind CSS + Framer Motion | 实现极致响应式适配与细腻流畅的动画交互。 |
| **视觉组件库** | **Aceternity UI** | 通过独特的科技感、未来感组件（如 Bento Grid, Spotlight），打破死板的蓝紫传统布局，建立极客美学。 |
| **后端服务** | **Go (Golang)** | 天生具备高并发能力，协程（Goroutine）极适合编写高频爬虫集群、高并发 API 调度与 AI 流水线编排。 |
| **动态任务调度** | Redis + **Asynq** | 借助 Asynq 任务队列在运行时动态创建、修改、取消用户的监控周期（如将24h灵活调整为6h），免于死板的固化 Cron 限制。 |
| **主数据库** | PostgreSQL | 关系型数据库，存储结构化的用户配置、监控关键词、通过验证的历史热点数据。 |
| **大模型接入** | **OpenRouter** | 统一大模型 API 适配器，动态无缝调用并随时切换最优模型。 |
| **大模型选型** | **DeepSeek v4 pro / DeepSeek-R1 系列** | 在保持极低 Token 成本的同时，提供抗衡行业顶尖水平的逻辑推理与内容去伪存真能力。 |

## 四、 AI 智能流水线与数据流转流程

### 1. 异构数据源接入
后端爬虫集群分为三类接口进行并发调度：
* **海外社交与技术源**：Twitter/X (指定 AI 意见领袖/官方号)、GitHub Trending & Commit Logs (捕获代码库突发变动)。*（部署时配置代理）*
* **国内主流社区源**：Bilibili (热门AI视频/专栏/动态)、深度技术社区。*（直连低延迟抓取）*
* **搜索引擎网络交叉验证**：Google Custom Search JSON API、Bing Web Search API。

### 2. 双阶段 AI 处理工作流 (Two-Stage Workflow)

```
[多源数据流 (推特/GitHub/B站)] ──> [Go 爬虫过滤器] ──> 触发关键词
                                                               │
                                                               ▼
[生成深度简报/实时推送] <── [DeepSeek推理裁判] <── [并发调用 Google/Bing 搜索比对]
```

* **阶段一：降噪与结构化（标准清洗）**
    * 利用 DeepSeek 标准版，对海量无序的文本进行去噪声（去除内嵌广告、无意义语气词）、语种统一翻译，并输出统一的 JSON 格式（标题、核心断言、原始链接、发布时间）。
* **阶段二：防伪推理裁判（深度思考）**
    * 当触发热点推送或关键词警报时，将阶段一的 JSON 数据连同 **Google/Bing 的前5条搜索摘要** 作为上下文，喂给 **DeepSeek 强推理模型**。
    * **Prompt 裁判准则**：
        1. *信源权威度评分*：是否有官方域名、Arxiv 论文或 GitHub 源码支撑？
        2. *反含糊炒作过滤*：文本是否包含缺乏数据支撑的极端营销词汇？
        3. *多源印证率*：国内源、海外源及搜索引擎的主流媒体报道之间能否形成逻辑闭环？
    * AI 最终输出：`{"is_authentic": true/false, "confidence_score": 0.95, "analysis": "..."}`。

## 五、 UI/UX 极客美学视觉视觉设计规范

拒绝千篇一律的传统蓝紫科技感，全面采用 **Graphite & Mint（石墨黑与荧光绿）** 的极简工业极客风，完美落地 Aceternity UI 核心组件：

1.  **公共看板 (10:00 Hotspot) -> Bento Grid (便当盒布局)**
    * 通过错落有致的非对称卡片展示 Top 5 热点。最核心、被 AI 验证最确凿的头条占据最大网格，配以 Aceternity 的微弱呼吸边框。点击卡片通过无缝的平滑过渡展开原始信源及 AI 判定报告。
2.  **个人监控面板 -> 3D Pin & Spotlight (三维悬浮与聚光灯)**
    * 用户输入的关键词卡片使用 `3D Pin` 效果，随鼠标悬停产生微弱的 3D 三维倾斜感。
    * 页面顶部 Hero 区域与关键数据高亮处采用 `Spotlight` 动态光束划过效果，鼠标移动时背景伴随极弱的科技粒子或光束（`Background Beams`），增强交互的高级感而绝不拖慢页面加载。
3.  **领域深挖阅读 -> Sticky Scroll Reveal (粘性滚动展现)**
    * 由于 24 小时或更长周期生成的简报内容较长，前端采用 `Sticky Scroll` 布局。随着用户向下滚动，左侧章节平滑固定，右侧的 AI 深度见解逐段显现（`Text Reveal`），让长文阅读更具段落感与仪式感，避免枯燥的长文本堆砌。
4.  **智能等待状态 -> Multi-step Loader (多步骤渐进加载)**
    * 当用户手动添加新关键词或系统正在通过 DeepSeek 进行防伪裁判时，前端展示 `Multi-step Loader`。
    * 依次平滑亮起：`[正在并发检索全网信源...]` -> `[正在调用Google/Bing进行交叉证实...]` -> `[DeepSeek防伪裁判深度推理中...]` -> `[验证通过/已生成简报]`。将后端的复杂流转转化为极具说服力的用户视觉体验。

## 六、 下阶段实施路径 (Roadmap)

1.  **第一阶段 (Milestone 1)**：搭建 Go 后端核心爬虫引擎，对接 OpenRouter 跑通 DeepSeek 提示词工作流；建立 PostgreSQL 与 Redis 基础环境。
2.  **第二阶段 (Milestone 2)**：开发 Next.js 前端，配置 Tailwind CSS 并基于 Context7 接入 Aceternity UI 组件，跑通 10:00 看板的数据渲染。
3.  **第三阶段 (Milestone 3)**：集成 Asynq 实现动态周期调度，接入 Google/Bing API 补全 AI 防伪裁判链路，进行黑盒压力测试。
4.  **第四阶段 (Milestone 4)**：全面功能验证正常后，基于上述沉淀的稳定 API，进一步封装和拓展 Agent Skills（如对接外部 AI Agent、提供 Webhook 回调能力）。
